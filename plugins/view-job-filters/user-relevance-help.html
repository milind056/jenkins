This filter includes/excludes jobs based on whether they contain the current user's ID or name somewhere.

Options

Search name:
This specifies whether to search jobs for the current user's ID or name or both. You can adjust the matching strategy by ignoring certain characters.
Match jobs:
What parts of a job to search for the current user's ID or name.
Build trend type:
How many builds that need to contain the current user's ID or name for this filter to match.
How far back to check:
A job can have many builds and this option determines how many of those builds will be checked. Always goes backwards from the latest build. Can be a decimal number, e.g. 1.5 hours. If you want to include all builds set this value to 0.
Match type:
Whether to add or remove jobs from this view. There's two kinds of match types:
  • Include matched/unmatched: adds additional jobs to this view.
  • Exclude matched/unmatched: removes jobs from this view that have been included by previous filters.
